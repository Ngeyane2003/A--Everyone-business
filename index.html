PayPal Integration Troubleshooting

Fix common issues with your PayPal integration

Common PayPal Integration Issues

Buttons Not Appearing

If your PayPal buttons aren't showing up at all, it's usually due to one of these issues:

  • Incorrect Client ID format or value
  • JavaScript errors on the page
  • PayPal SDK not loading properly
  • Container element not existing when script runs

Buttons Appear But Don't Work

If buttons appear but nothing happens when clicked:

  • Server endpoints not set up correctly
  • CORS issues with API requests
  • Incorrect API response format

Payment Fails

If payments are failing during processing:

  • Server-side validation errors
  • Currency or amount format issues
  • PayPal account configuration problems

Step-by-Step Troubleshooting

1

Check Browser Console for Errors

Open your browser's developer tools (F12) and look for any JavaScript errors in the Console tab.

// Common errors to look for: - "PayPal is not defined" (SDK loading issue) - "Cannot read property 'Buttons' of undefined" - CORS errors related to API calls
2

Verify Your Client ID

Ensure you're using the correct Client ID for your environment (sandbox vs live).

// In your HTML head: <script src="https://www.paypal.com/sdk/js?client-id=YOUR_CLIENT_ID¤cy=USD"> </script>

Make sure to replace YOUR_CLIENT_ID with your actual live client ID from the PayPal Developer Dashboard.

3

Check Server Endpoints

Your PayPal buttons are configured to call serverless functions at these endpoints:

// Create order endpoint POST /api/create-order // Capture order endpoint POST /api/capture-order

Make sure these endpoints exist on your server and return the expected JSON responses.

4

Test with a Simple Example

Try a basic PayPal integration first to isolate the issue:

paypal.Buttons({ createOrder: function(data, actions) { return actions.order.create({ purchase_units: [{ amount: { value: '1.00' } }] }); }, onApprove: function(data, actions) { return actions.order.capture().then(function(details) { alert('Transaction completed by ' + details.payer.name.given_name); }); } }).render('#paypal-button-container');

If this works, your issue is with the server integration.

Vercel Serverless Functions

Setting Up API Endpoints on Vercel

For your PayPal integration to work, you need to create serverless functions:

1

Create API Directory

In your Vercel project, create an /api directory at the root level.

2

Create Order Endpoint

Create /api/create-order.js:

const paypal = require('@paypal/checkout-server-sdk'); // Set up PayPal environment const environment = new paypal.core.SandboxEnvironment( process.env.PAYPAL_CLIENT_ID, process.env.PAYPAL_CLIENT_SECRET ); const client = new paypal.core.PayPalHttpClient(environment); module.exports = async (req, res) => { if (req.method !== 'POST') { return res.status(405).end(); } try { const request = new paypal.orders.OrdersCreateRequest(); request.requestBody({ intent: 'CAPTURE', purchase_units: [{ amount: { currency_code: 'USD', value: req.body.value || '1.00' }, description: req.body.description || 'Test Payment' }] }); const order = await client.execute(request); res.status(200).json({ id: order.result.id }); } catch (error) { console.error('Create order error:', error); res.status(500).json({ error: error.message }); } };
3

Capture Order Endpoint

Create /api/capture-order.js:

const paypal = require('@paypal/checkout-server-sdk'); // Set up PayPal environment (same as above) const environment = new paypal.core.SandboxEnvironment( process.env.PAYPAL_CLIENT_ID, process.env.PAYPAL_CLIENT_SECRET ); const client = new paypal.core.PayPalHttpClient(environment); module.exports = async (req, res) => { if (req.method !== 'POST') { return res.status(405).end(); } try { const { orderID } = req.body; const request = new paypal.orders.OrdersCaptureRequest(orderID); request.requestBody({}); const capture = await client.execute(request); res.status(200).json(capture.result); } catch (error) { console.error('Capture order error:', error); res.status(500).json({ error: error.message }); } };
4

Set Environment Variables

In your Vercel project settings, set these environment variables:

PAYPAL_CLIENT_ID=your_client_id_here PAYPAL_CLIENT_SECRET=your_client_secret_here

Never commit your client secret to version control! Always use environment variables.

Testing Your Integration

Test with the PayPal Sandbox

Always test with sandbox credentials before going live:

  • Use sandbox client ID for testing
  • Create test buyer and merchant accounts in PayPal Developer Dashboard
  • Test successful and failed payment scenarios
Create Sandbox Accounts

Verify Server Responses

Check that your server endpoints return the correct responses:

// create-order should return: { id: "ORDER_ID" } // capture-order should return: { id: "CAPTURE_ID", status: "COMPLETED" }

Use browser dev tools Network tab to inspect API calls.

Check Payment Status in PayPal Dashboard

After a test transaction, verify it appears in your PayPal dashboard:

  • Sandbox transactions appear in the developer dashboard
  • Live transactions appear in your business account
  • Look for any error codes or declined transactions
